# 课程核心模型

## 核心抽象

课程不要按行业写死，而是抽象为：

```text
课程产品 -> 班级实例 -> 课次执行 -> 考勤/作业/测评/结课
```

## 课程产品 CourseProduct

表示机构售卖或教学规划中的课程。

核心字段：

- 租户 ID
- 课程名称
- 课程分类
- 授课方式：线下、直播、录播、混合
- 计费方式：按课时、按期、按课包、按月
- 总课时
- 标准价格
- 扩展模板
- 上下架状态

## 班级实例 ClassGroup

表示某个课程实际开出的班。

核心字段：

- 课程产品 ID
- 校区 ID
- 班级名称
- 班主任
- 容量
- 开班日期
- 结班日期
- 班级状态

## 课次 LessonSession

表示具体一节课。

核心字段：

- 班级 ID
- 老师 ID
- 教室 ID
- 线上房间地址
- 计划开始/结束时间
- 实际开始/结束时间
- 状态

## 扩展字段

行业差异用扩展模板处理：

```text
文化课：年级、学科、教材版本、考试类型
艺术类：级别、考级、作品集、器材
线上课：直播平台、回放地址、设备要求
线下课：教室、座位、耗材
```

建议新增：

```text
extension_template
extension_field
business_extension_value
```

这样每个租户可以配置自己的课程字段，而不用频繁改表结构。
